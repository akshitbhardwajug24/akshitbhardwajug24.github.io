01 Theoretical Background
Chomsky Normal Form
CNF
A context-free grammar is in CNF if every production rule takes one of exactly two forms: a non-terminal producing exactly two non-terminals, or a non-terminal producing exactly one terminal symbol. This form is fundamental to the CYK parsing algorithm, which runs in O(n³) time.
A → B C   // two non-terminals
A → a       // one terminal
S → ε       // only if S is start
Greibach Normal Form
GNF
A grammar is in GNF if every production starts with exactly one terminal symbol followed by zero or more non-terminals. GNF is useful because it directly corresponds to top-down parsing: each derivation step consumes exactly one input terminal, enabling construction of LL(1) parsers.
A → a B C  // terminal then NTs
A → a       // terminal alone
A → b B     // terminal + one NT
CNF — 5 Transformation Steps
  • 1Add a new start symbol S₀ → S if the start symbol appears in any right-hand side.
  • 2Compute nullable non-terminals and eliminate all ε-productions by expanding combinations.
  • 3Remove unit productions A → B by computing the unit closure and inlining their rules.
  • 4For terminals in long productions, introduce Tₐ → a and substitute the terminal.
  • 5Binarize productions of length ≥ 3 into chains of binary rules using new non-terminals.
GNF — 2 Additional Steps
  • 6Eliminate all left recursion (direct and indirect) using the A/A′ method with a fixed non-terminal ordering.
  • 7Work from the highest-indexed non-terminal downward, substituting leading non-terminals until every production begins with a terminal.
Why these two forms?
CNF is used in CYK parsing and proof of the pumping lemma. GNF simplifies construction of pushdown automata — each rule pops exactly one symbol per derivation step.
